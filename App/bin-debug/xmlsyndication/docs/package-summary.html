All Packages


The XML Syndication project provides simple APIs for parsing RSS .91, .92, 1.0, and 2.0 XML documents, as well as Atom 1.0 XML documents. It also provides a way to generically parse and access any type of RSS or Atom document without having to know what kind of document it is (useful for projects like news aggregators).

There are three packages in the XML Syndication project: rss, atom, and generic. The rss packages provides APIs for parsing all versions of RSS, the atom package provides APIs for parsing Atom 1.0, and the generic package provides APIs for automatically determining what kind of XML document you have, parsing it, and giving you a generic way to access data from it. By generic, I mean that the generic package uses APIs that are higher level than both the rss and atom packages, allowing you to access data which is more or less common or equivalent across all versions of RSS and Atom 1.0.

None of the parsers strictly enforce any of the RSS or Atom specifications. They support almost everything in all the specifications in the sense that they can be used to access anything that might be present in any type of feed, however they don't require the feeds to strictly comply with the specifications. They parse and allow you to access everything that is there, regardless of dependencies, and whenever something is not there, whether the specification indicates that it's required or not, the APIs will return null for Strings or NaN for numbers. The parsers are therefore comprehensive, but not strict, which allows them to parse the largest possible number of feeds. The feeds are required to be well-formed, however. Although the XML Syndication package is forgiving with respect to the RSS and Atom specifications, it is strict with respect to the XML specification.

For parsing RSS .91, .92, and 2.0 documents, use the RSS20 class in the rss package. Since RSS 2.0 is backward-compatible with .91 and .92, the RSS20 class will successfully parse them all. For RSS 1.0 documents, use the RSS10 class (RSS 1.0 is not closely related to .91, .92, or 2.0). Wherever there is overlap between RSS 1.0 and RSS 2.0, the rss package uses interfaces to enable you to treat them as though they were the same. For instance, since all versions of RSS have a concept of a channel, you can access both RSS 1.0 and RSS 2.0 channels using the "IChannel" interface in a more generic way.

For parsing Atom 1.0 documents, use the Atom10 class in the atom package. Since the Atom 1.0 specification has not been finalized, it is not 100% comprehensive, however it will definitely successfully parse the overwhelming majority of Atom feeds out there, and will definitely allow you to get at the core information of all of them.

If you don't know what kind of feed you want to parse, you can use the FeedFactory class in the generic package. It will figure out what kind of feed you gave it, and will return an object that implements the IFeed interface. That means you can access data that is common or equivalent across all versions of RSS and Atom without having to know what kind of feed you have. Of course, you give up some of the more fine-grained aspects of each specification, but you can get at all the most important information. If you hand the FeedFactory class a feed it doesn't recognize, it will throw an UnknownFeedError.
